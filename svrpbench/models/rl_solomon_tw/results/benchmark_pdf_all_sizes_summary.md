# PDF 同口径静态 TWCVRP 全规模评估汇总

评估口径：
- 数据：`data_splits/universal_v1/test/{100,200,400,600,800,1000}`。
- 模式：`static`，不含交通扰动。
- 成本：`avg_cost = total_travel_time + waiting_time`。
- 静态时间矩阵：欧氏距离四舍五入为整数。
- 时间推进：不加入 Solomon `service_time`，与 PDF/OR-Tools 评估代码保持一致。
- CVR：`100 * (time_window_violations + capacity_violations) / num_customers`。

## 结论摘要

- 新结果已使用完全 PDF 同口径评估；Solomon `service_time` 不参与时间推进和成本计算。
- 两个 RL 模型在所有规模上均达到 `CVR=0.0%`、`可行率=100%`、`车辆超限=0`。
- 100-800 规模下，两个 RL 模型的平均总成本均低于 PDF 表格；1000 规模下，两个 RL 模型平均总成本略高于 PDF 表格。
- 平均客户成本随规模总体上升，主要来自等待时间增长；大规模下等待占比稳定在约 `58%-62%`。
- `static100` 在 200/400/1000 上略优于 universal；100 规模 universal 略优；600/800 二者几乎持平。
- universal 当前优势不是成本全面领先，而是单个 checkpoint 覆盖多规模，并保持可行性稳定。
- 与 PDF 表格相比，RL 推理时间显著更低；但该时间只代表训练后模型推理和 decoder 求解，不包含训练成本。
