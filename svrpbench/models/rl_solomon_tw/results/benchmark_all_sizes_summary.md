# PDF 同口径静态 TWCVRP 全规模评估汇总

评估口径：

- 数据：`universal_v1` 测试口径；100 使用 `benchmark_universal100_test.csv`，200-1000 使用对应 `benchmark_{size}_test.csv`。
- 模式：`static`，不含交通扰动。
- 成本：`avg_cost = total_travel_time + waiting_time`。
- CVR：`100 * (time_window_violations + capacity_violations) / num_customers`。
- `static100` 在 200-1000 上属于跨规模泛化测试；`universal` 是多规模通用模型。

## 结论摘要

- 两个模型在全部规模上均达到 `CVR=0.0%`、`可行率=100%`，说明当前 `strict_insert` 可行性优先策略在静态 TWCVRP 上非常稳定。
- `static100` 在所有规模上的平均总成本均低于 `universal`，但 600/800 规模差距很小，分别约为 `0.4%` 与 `0.5%`。
- `universal` 的核心价值不是在 100-1000 每个规模上都压低单规模成本，而是一个 checkpoint 覆盖所有规模。
- 随规模增大，等待占比明显升高，600-1000 规模约接近或超过 `48%`，说明大规模实例中的时间窗等待成为主要成本来源。
- 求解时间随规模上升，但仍保持在秒级；1000 规模约 `2.09-2.11s`。
